# Plan 18 — Sessions HTTP API

## Summary

Expose the sessions store via two read-only JSON endpoints on the existing query API listener (`:9090`): `GET /api/v1/sessions?since=<duration>` and `GET /api/v1/sessions/{id}/events?since=<duration>&limit=<n>`. Path-matched via a small sub-router so the Prometheus API code stays untouched.

## Dependencies

- **Plan 15** (sessions store)
- **Plan 17** (server wiring — the bbolt store is opened in main.go)

## Scope

### In Scope

- `internal/query/sessions_api.go`:
  - `SessionsAPI` struct with `logger`, `store sessions.Store`
  - `NewSessionsAPI(logger *slog.Logger, store sessions.Store) *SessionsAPI`
  - `func (s *SessionsAPI) Register(mux *http.ServeMux)` — registers:
    * `mux.HandleFunc("/api/v1/sessions", s.listSessions)` — strict match (rejects subpaths to avoid colliding with the next route)
    * `mux.HandleFunc("/api/v1/sessions/", s.sessionSubpath)` — parses `{id}/events`
  - `listSessions(w, r)`:
    * `since` query param — parsed by `time.ParseDuration`; default `5m`; max `24h`
    * Compute `cutoff := time.Now().Add(-since)`
    * Call `store.ListSessions(r.Context(), cutoff)` and JSON-encode
    * Response shape: `{"status":"success","data":[Session,...]}`
  - `sessionSubpath(w, r)`:
    * Trim prefix `/api/v1/sessions/`; split on `/`
    * Validate `[sessionID, "events"]`; reject other shapes with 404
    * Query params: `since` (default `1h`), `limit` (default `100`, max `1000`)
    * Call `store.GetEvents(r.Context(), sessionID, cutoff, limit)`
    * Response shape: `{"status":"success","data":[Event,...]}`
  - 404 on unknown session: when `GetEvents` returns 0 events AND `GetSession` errors → JSON 404
- `internal/query/sessions_api_test.go`:
  - Use a moq `sessions.Store` (already generated by Plan 15)
  - Cases:
    * `GET /api/v1/sessions` with default since → 200, JSON contains store's mocked list
    * `GET /api/v1/sessions?since=1h` → 200, cutoff passed to store equals `now-1h ± 1s` (use `cmp.WithinDuration`-style check)
    * `GET /api/v1/sessions?since=garbage` → 400
    * `GET /api/v1/sessions/abc/events?limit=50` → 200, store called with id `abc` and limit 50
    * `GET /api/v1/sessions/abc/events?limit=99999` → store called with capped limit `1000`
    * `GET /api/v1/sessions/abc/extras` → 404
    * `GET /api/v1/sessions/abc/events` with unknown id (store returns 0 events + GetSession ErrNotFound) → 404 JSON
- `cmd/server/main.go` change:
  - Pass `sessionStore` into `startQueryAPI`
  - Inside `startQueryAPI`, after creating the Prometheus API mux, register sessions endpoints on the *same* outer mux:
    ```go
    if sessionStore != nil {
        sa := query.NewSessionsAPI(queryLogger.With("component", "sessions-api"), sessionStore)
        sa.Register(mux)
    }
    ```
  - The sessions handlers must be registered on the outer mux (the one that also has `/healthz`), not on `api.Handler()`'s internal mux — easier to do without touching existing code
- CORS: the sessions endpoints inherit the existing `corsMiddleware` if registered above it. To stay consistent, wrap the registered sessions handlers with the same `corsMiddleware` already applied to `api.Handler()`. Simplest: register `corsMiddleware(sessionsMux)` on the outer mux at `/api/v1/sessions` and `/api/v1/sessions/`.

### Out of Scope

- WebSocket / SSE — polling only for v1
- Pagination cursors — `limit` is enough for v1
- Filtering by event name or tool — clients filter client-side

## Acceptance Criteria

1. `go build ./...` succeeds
2. `go test ./internal/query/...` passes including the new test cases
3. `make lint` clean
4. With the server running and a session populated:
   - `curl localhost:9090/api/v1/sessions?since=10m` returns JSON `{status:"success",data:[{id,user_id,model,...}]}`
   - `curl localhost:9090/api/v1/sessions/<id>/events?limit=5` returns the 5 most recent events
5. Unknown session id returns HTTP 404 with a JSON body `{status:"error",error:"session not found"}`
6. When the sessions store is disabled (`cfg.Logs.Enabled=false`), `/api/v1/sessions` returns 404 (because the handler is never registered) — existing Prometheus endpoints unaffected
