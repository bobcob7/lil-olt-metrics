# internal/store

## Purpose

Pluggable storage backends for metric time series data. Provides a common interface consumed by both the ingestion and query subsystems, plus a Prometheus adapter for PromQL engine compatibility.

## Functionality

- `Store` interface: write (via Appender), select, label names/values, close
- `Appender` interface: batched sample writing with commit/rollback
- `SeriesSet` / `Series` / `Sample` types for internal iteration
- Prometheus `storage.Queryable` adapter wrapping any Store for PromQL engine use
- `moq`-generated mocks for Store and Appender

## Files

| File | Description |
|------|-------------|
| `interfaces.go` | Store, Appender, SeriesSet, Series, Sample type definitions + moq directive |
| `adapter.go` | Prometheus storage.Queryable/Querier adapter and SeriesSet bridge |
| `adapter_test.go` | Unit tests for the adapter layer using stub Store |
| `moq_test.go` | Generated mocks for Store and Appender |
