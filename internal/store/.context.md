# internal/store

## Purpose

Pluggable storage backends for metric time series data. Provides a common interface consumed by both the ingestion and query subsystems.

## Functionality

- Common `Store` interface for write and read operations
- Built-in FS storage engine (WAL, head block, persistent blocks, compaction)
- Prometheus remote write/read backend
- VictoriaMetrics remote backend
- Implements Prometheus `storage.Queryable` for PromQL engine compatibility

## Files

| File | Description |
|------|-------------|
| `interfaces.go` | Store interface definition and shared types |
| `fs.go` | Built-in filesystem storage engine |
| `prometheus.go` | Prometheus remote write/read backend |
| `victoriametrics.go` | VictoriaMetrics remote backend |
