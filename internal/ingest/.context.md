# internal/ingest

## Purpose

Handles OTLP metrics ingestion over gRPC and HTTP, translating the OTLP metric model into Prometheus-compatible time series for storage.

## Functionality

- OTLP gRPC handler implementing MetricsService/Export
- OTLP HTTP handler accepting POST /v1/metrics (protobuf and JSON via protojson)
- gzip decompression for HTTP requests
- OTLP-to-Prometheus metric translation (Gauge, Sum, Histogram, ExponentialHistogram, Summary)
- Delta-to-cumulative conversion with TTL-based eviction for bounded memory
- Resource attribute mapping (configurable label map + promote list)
- Scope name/version as `otel_scope_name`/`otel_scope_version` labels
- Schema URL recording as `__schema_url__` label (configurable)
- Metric name sanitization for Prometheus compatibility
- Unit suffix and type suffix handling
- Exemplar extraction with trace/span ID preservation
- Partial success reporting when some data points fail translation
