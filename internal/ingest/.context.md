# internal/ingest

## Purpose

Handles OTLP metrics ingestion over gRPC and HTTP, translating the OTLP metric model into Prometheus-compatible time series for storage.

## Functionality

- OTLP gRPC handler implementing MetricsService/Export
- OTLP HTTP handler accepting POST /v1/metrics (protobuf)
- gzip decompression for HTTP requests
- OTLP-to-Prometheus metric translation (Gauge, Sum, Histogram)
- Resource attribute mapping (configurable label map + promote list)
- Scope name/version as `otel_scope_name`/`otel_scope_version` labels
- Metric name sanitization for Prometheus compatibility
- Unit suffix and type suffix handling
- Exemplar extraction with trace/span ID preservation
- Partial success reporting when some data points fail translation

## Files

| File | Description |
|------|-------------|
| `interfaces.go` | Consumer interfaces for translator and store, moq generate directive |
| `grpc.go` | OTLP gRPC handler (MetricsService/Export) |
| `grpc_test.go` | gRPC handler tests (success, empty, partial, errors) |
| `http.go` | OTLP HTTP handler (POST /v1/metrics, protobuf) |
| `http_test.go` | HTTP handler tests (success, gzip, malformed, partial, errors) |
| `translator.go` | OTLP-to-Prometheus translator, writes to store.Appender |
| `translator_test.go` | Unit tests for each metric type, labels, sanitization |
| `moq_test.go` | Generated mocks for metricsTranslator and metricsStore |
